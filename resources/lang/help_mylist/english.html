TYPES OF LOGIC UNION :

AND: The logical union 'AND' work by joining two or more filters when when conditions between these filters are met in each.

OR: The logical union 'OR' works by joining two or more filters when the conditions between these filters are met in at least one.

TYPES OF OPERATORS FOR A FILTER:

LIKE:The like operator works exclusively with fields of type string to find matches for the search text in the column to be affected. You can use the wildcard '%' to facilitate searching. The wildcard '%' works as follows: (% abc) to coincide at the end of the field abc (abc%) to coincide abc at the beginning of the field (% abc%) Find the match anywhere in the abc field where the field also refers to the affected column.

IN: The IN operator is a special type of operator that searches multiple values ​​in a selected field. This operator can work with fields string and numeric types. To use it we can do as follows, for example find records that contain the data given column 1, 2 and 3, the right way to do serious writing 1,2,3 in the value field, thus the filter when applied to seek the records in the selected field are equal to 1 or 2 or 3.

> GREATER THAN TO: Look for records that are greater than written value and only works with fields of type numeric.

>= GREATER THAN OR EQUAL TO: Looking for records that are greater than or equal to the value written and only works with numeric type fields.

= EQUAL: Find records that are equal to the value written and works with fields of type numeric or string type.

<> DIFFERENT: Find records that are different of the written value and works with fields of type numeric or string type.

<= LESS THAN OR EQUAL TO: Find records that are less than or equal than writing value, only works with numeric type fields.

< LESS THAN TO: Search records that are less than the value written and only works with numeric type fields.

NOT IN: It is the opposite condition of IN.

NOT LIKE: It is the opposite condition LIKE.
